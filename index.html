Behaviour Driven Automation (BDA)
Feature
Drive web automation with standard Gherkin feature specs like this:
  Feature: Google search

 Scenario: Lucky Google search
     Given I have Google in my browser                    (1)
      When I do a search for "Gwen automation"            (2)
      Then I should find a Gwen page                      (3)
Meta
..by defining step definitions in separate Gherkin meta specs like this:
  Feature: Google search meta
                        
 @StepDef
 Scenario: I have Google in my browser                    (1)
     Given I start a new browser
      When I navigate to "http://www.google.com"
      Then the page title should be "Google"
                        
 @StepDef
 Scenario: I do a search for "<query>"                    (2)
     Given the search field can be located by name "q"
      When I enter "$<query>" in the search field
      Then the page title should contain "$<query>"
                                               
 @StepDef
 Scenario: I should find a Gwen page                      (3)
     Given link 1 can be located by css selector ".r > a"
      When I click link 1
      Then the current URL should match regex ".+[G|g]wen.*"
Gwen
Launch Gwen to bind the meta and execute the feature. That's it!
