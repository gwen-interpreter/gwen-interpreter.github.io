# DSL Steps

All Gherkin steps that match the DSL expressions here are readily executable by Gwen. The DSL matched is the operation that will be performed when a step is interpreted for execution.

## Page navigation and URLs

#### Given..

<details id="the-page-url-is-url">
  <summary class="dsl">

  ```gherkin
  the <page> url is "<url>"
  ```
  <p>Puts the given page into scope and binds the given URL to it.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;page&gt;</code> is the name of the page</li>
    <li><code>&lt;url&gt;</code> is the page URL (can be in <Link to="https://github.com/gwen-interpreter/gwen/wiki/Doc-Strings">DocString</Link> position)</li>
  </ul>
  <p>Example</p>

  ```gherkin {1}
    Given the todo page url is "https://todomvc.com/examples/react"
      And the todo field can be located by class "new-todo"
  ```

<details id="i-am-on-the-page">
  <summary class="dsl">


  <summary class="dsl">

<details id="i-capture-the-text-in-the-current-url-by-regex-as-name">
  <summary class="dsl">

  ```gherkin
  I capture the text in the current URL by regex "<expression>" as <name>
  ```

  <p>Extracts a substring from the current browser URL using the given regex and binds it to the given name in the global scope.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;expression&gt;</code> is the regex expression</li>

  </ul>
  <p>Example</p>


#### Then..

<details id="the-current-url-should-match-expression">
  <summary class="dsl">

  ```gherkin
  the current URL should[ not] <match> "<expression>"
  ```
  
  <p>Checks whether or not the current browser URL matches a given expression. Reports an assertion error if the match fails.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>not</code> negates the match if included</li>
    <li><code>&lt;match&gt;</code> is the type of match to perform, one of:
      <ul>
        <li><code>be</code> for exact match</li>
        <li><code>contain</code> for partial match</li>
        <li><code>start with</code> for partial leading match</li>
        <li><code>end with</code> for partial trailing match</li>
        <li><code>match regex</code> for regex match</li>
        <li><code>match xpath</code> for XML match</li>
        <li><code>match template</code> for <Link to="https://github.com/gwen-interpreter/gwen/wiki/Template-Matching">template</Link> match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
        <li><code>match template file</code> for <Link to="https://github.com/gwen-interpreter/gwen/wiki/Template-Matching">template</Link> file match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
      </ul>
    </li>
    <li><code>&lt;expression&gt;</code> is the expression to match (can be in <Link to="https://github.com/gwen-interpreter/gwen/wiki/Doc-Strings">DocString</Link> position)</li>
  </ul>
  <p>Example</p>

  ```gherkin {2,3}
    When I navigate to "https://todomvc.com/examples/react"
    Then the current URL should contain "todomvc.com"
     And the current URL should not contain "other.com"
  ```

    <div align="right">Since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v1.0.0">v1.0.0</Link></div>
  </div>
</details>

        <li><code>start with</code> for partial leading match</li>
        <li><code>end with</code> for partial trailing match</li>
        <li><code>match regex</code> for regex match</li>
        <li><code>match xpath</code> for XML match</li>
        <li><code>match template</code> for <Link to="https://github.com/gwen-interpreter/gwen/wiki/Template-Matching">template</Link> match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
        <li><code>match template file</code> for <Link to="https://github.com/gwen-interpreter/gwen/wiki/Template-Matching">template</Link> file match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>

    </li>
    <li><code>&lt;expressionRef&gt;</code> is the name of the binding containing the expression to match</li>

  ```gherkin {4,5}
      And some other domain is "someother.com"

     Then the current URL should contain my domain
      And the current URL should not contain some other domain
  ```

## Web element selectors

#### Given..

<details id="element-can-be-located-by-selector-expression">
  <summary class="dsl">

  ```gherkin
  <element> can be located by <selector> "<expression>"
  ```

  <p>Binds a selector for a web element to the current scope with a default timeout.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;element&gt;</code> is the web element name</li>
    <li><code>&lt;selector&gt;</code> is the selector type, one of:
      <ul>
        <li><code>id</code> for selecting by id attribute</li>
        <li><code>name</code> for selecting by name attribute</li>
        <li><code>tag name</code> (or <code>tag</code> since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.46.0">v2.46.0</Link>) for selecting by tag name</li>
        <li><code>css selector</code> (or <code>css</code> since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.46.0">v2.46.0</Link>) for selecting by css expression</li>
        <li><code>class name</code> (or <code>class</code> since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.46.0">v2.46.0</Link>) for selecting by class attribute</li>
        <li><code>xpath</code> for selecting by XPath</li>
        <li><code>link text</code> for selecting by link text</li>
        <li><code>partial link text</code> for selecting by partial link text</li>
        <li><code>javascript</code> (or <code>js</code> since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.46.0">v2.46.0</Link>) for selecting <Link to="https://gweninterpreter.wordpress.com/2019/08/13/locating-web-elements-with-gwen/#byJavaScript">by JavaScript</Link></li>
      </ul>
    </li>
    <li><code>&lt;expression&gt;</code> is the selector expression</li>
  </ul>
  <p>Example</p>

  ```gherkin {1}
    Given the todo field can be located by class "new-todo"
     When I enter "Walk the dog" in the todo field
  ```
