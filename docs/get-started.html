Getting Started

The quickest and easiest way to get started with Gwen is to initialise your project directory with the standard Gwen file system setup. Be sure to install Gwen before proceeding to do so.

Initialising your project directory#

  • npm
  • Yarn
  • Standalone

Run the following command in the root of your JS project where you installed Gwen as a dev dependency:

npm run gwen init

Set the default browser to use#

Gwen is configured to use the Chrome browser by default. You can change this default to another browser by amending the default gwen.cli.options.config setting in the gwen.conf file in your project directory.

gwen {  ..  cli {    options {      ..      config = [        "gwen/browsers/chrome.conf", <<-- Amend this to your preferred browser conf        "gwen/env/local.conf"      ]      ..    }  }  ..}

Run the Gwen samples#

Run the following command in your project directory to execute the sample features created by the init command you ran earlier to initialise your directory.

  • npm
  • Yarn
  • Standalone
npm run gwen -b gwen/samples
yarn gwen -b gwen/samples
gwen -b samples
note

The -b (or --batch) switch tells Gwen to exit immediately after executing the samples. Omitting it will open the Gwen REPL console instead of exiting after the samples have executed.

Spend some time studying the samples and launching Gwen several times and have a look at the HTML evaluation reports generated in the output directory too.

You can keep the samples or delete them once you're done (up to you).

Share your project with your team#

Once you're confortable with all of the above, you might want to push your project to a Git repository and share it with your team. A .gitignore file that excludes the output directory is included.
