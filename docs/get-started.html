Get Started

Prerequisites#

  • Linux, Mac or Windows OS
  • Chrome, Firefox, Safari, or Edge or IE web browser
  • Java SE 8+
    • In order to run Gwen, you need to have a Java runtime environment installed. You can check by running java -version in a terminal window. A version will be displayed if you do. If not, install Java.
Before you start

Read the Gwen introduction.

Project setup#

  • Standalone Project
  • Workspace Project
  • JS Project

1. Download and install#

Perform the following to set up Gwen as a standalone project in a directory on your machine:

  • Browse the releases and download the binary gwen-web-version.zip package of the version you want to install and use
  • Extract the downloaded ZIP into a desired directory

This will give you the following files and subdirectories:

No Automatic Updates

There are no automatic updates with the standalone approach. You will need to repeat this process when you want to upgrade or downgrade your installed Gwen verison. You will also need to do the same on your build server(s) if you plan to run Gwen there.

2. Set your default browser#

Gwen is configured to use the Chrome browser by default. To change it, create a new gwen.properties file in your standalone project directory and set the following setting to chrome ,firefox, safari, edge, or ie:

File: gwen.properties

# default browser to usegwen.web.browser = firefox

(This example sets FireFox as the default browser.)

3. Execute the samples#

Run the following command in your project root to execute the sample feature specs. An evaluation report will be generated at target/reports/index.html in your project.

  • Linux
  • Windows
./gwen -b -r target/reports samples
gwen -b -r target/reports samples
Batch or REPL
  • The -b switch instructs Gwen to exit immediately after executing the samples. Omitting it will open the REPL instead.
  • The -r option specifes the report output directory

4. Add your own features and meta#

Create a new features subdirectory in your project root and start adding your own features and meta there. Run the following command to execute your features:

  • Linux
  • Windows
./gwen -b -r target/reports features
gwen -b -r target/reports features

2. Enable/Disable automatic updates#

When you first launch Gwen in your workspace, it will take a little time to download the latest version of itself before starting. On subsequent launches it will start much faster and won't download anything again until we publish a new release in which case it will automatically update itself.

To turn off automatic updates, open the gwen.properties file in your workspace project directory in a text editor and amend the gwen.gwen-web.version property from latest to the version number of the release you want to use.

# Gwen package version# --------------------# Gwen web version (specify desired version or latest for auto updates)gwen.gwen-web.version=latest

3. Set your default browser#

Gwen is configured to use the Chrome browser by default. To change it, open the Gwen launch script in your workspace project directory and set the follwing variable to chrome, firefox, safari, edge, or ie:

  • Linux
  • Windows

File: gwen

DEFAULT_BROWSER="firefox"

File: gwen.bat

SET DEFAULT_BROWSER=firefox

(This example sets FireFox as the default browser.)

4. Execute the samples#

Run the following command in your project root to execute the sample feature specs. An evaluation report will be generated at target/reports/index.html in your project.

  • Linux
  • Windows
./gwen -b samples
gwen -b samples
Batch or REPL
  • The -b switch instructs Gwen to exit immediately after executing the samples. Omitting it will open the REPL instead.

5. Add your own features and meta#

Start adding your own features and meta to the features directory in your workspace. Run the following command to execute your features:

  • Linux
  • Windows

To use the default browser and env/local.properties settings:

./gwen -b features

To use the Edge browser and env/local.properties settings:

./gwen edge -b features

To use the default browser and env/dev.properties settings:

./gwen dev -b features

To use the Edge browser and env/dev.properties settings:

./gwen edge dev -b features

To use the default browser and env/local.properties settings:

gwen -b features

To use the Edge browser and env/local.properties settings:

gwen edge -b features

To use the default browser and env/dev.properties settings:

gwen dev -b features

To use the Edge browser and env/dev.properties settings:

gwen edge dev -b features

6. Share your project with your team#

Push your project to a Git repository and share it with your team. A .gitignore file that excludes the target output directory is included. You can keep or delete the samples subdirectory (up to you).
