# Annotations

:::tip 
Use annotations in [meta](/docs/meta) and avoid them in features where possible.
:::

| Annotation                                                    | Level(s)                                                           | Description                                                                                                                          |
| :------------------------------------------------------------ | :----------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------- |
| [`@StepDef`](/docs/meta#stepdefs)                             | Scenario                                                           | Turns a Gherkin Scenario into a callable step definition                                                                             |
| [`@Import`](/docs/meta/load-strategies#import-strategy)       | Feature<br/>Meta                                                   | Loads only specified meta files into a feature (or another meta) instead relying on the default loading mechanisms                   |
| [`@Synchronized`](/docs/meta#synchronized-stepdefs)           | StepDef                                                            | Ensures that only one thread can call a StepDef at any one time during parallel execution                                            |
| [`@Synchronised`](/docs/meta#synchronized-stepdefs)           | StepDef                                                            | Same as a `@Synchronized` above but with AU spelling instead                                                                         |
| [`@DataTable`](/docs/advanced/data-tables)                    | StepDef                                                            | Associates a step definition with a Gherkin data table                                                                               |
| [`@ForEach`](/docs/advanced/data-tables)                      | StepDef                                                            | Interates over the records of a Gherkin data table                                                                                   |
| [`@Examples`](/docs/modes/data-driven#examples-annotation)    | Scenario Outline                                                   | Imports data from a CSV file into an examples table in a Scenario Outline                                                            |
| [`@Context`](/docs#behavioural-rules-and-semantics)           | StepDef                                                            | Assigns *context* behavior to a step definition so it can only be called by *Given* steps                                            |
| [`@Action`](/docs#behavioural-rules-and-semantics)            | StepDef                                                            | Assigns *action* behavior to a step definition so it can only be called by *When* steps                                              |
| [`@Assertion`](/docs#behavioural-rules-and-semantics)         | StepDef                                                            | Assigns *assertion* behavior to a step definition so it can only be called by *Then* steps                                           |
| [`@Breakpoint`](/docs/cli/repl#breakpoints)                   | Step                                                               | Pauses execution on a step when Gwen is launced in debug mode                                                                        |
| [`@Finally`](/docs/advanced/finally-steps#finally)            | Step                                                               | Forces the last step in a Scenario to always execute even when prior ones fail or are skipped                                        |
| [`@Try`](/docs/advanced/try-steps)                            | Step                                                               | Instructs Gwen to ignore a failed step and continue procesing                                                                        |
| [`@Eager`](/docs/advanced/eager-or-lazy#eager)                | Step                                                               | Immediately evaluates and assings values to refereces in place instead of at each call site                                          |
| [`@Lazy`](/docs/advanced/eager-or-lazy#lazy)                  | Step                                                               | Evaluates and assings values to refereces when they are first used at a call site only                                               |
| [`@Hard`](/docs/advanced/assertion#hard-assertions-default)   | Step                                                               | Causes a failed assertion step to halt further processing before reporting failure                                                   |
| [`@Soft`](/docs/advanced/assertion#soft-assertions)           | Step                                                               | Causes a failed assertion step to continue processing next steps before reporting failure                                            |
| [`@Sustained`](/docs/advanced/assertion#sustained-assertions) | Step                                                               | Causes a failed assertion step to continue processing next steps without reporting failure                                           |
| [`@DryRun`](/docs/modes/dry-runs#dryrun-annotations)          | Step                                                               | Forces references to take on certain given values when evaluated in dry runs                                                         |
| `@Message`                                                    | Step                                                               | Overrides the default error message for a failed [assertion](/docs/dsl/reference#text-ref-should-match-expression) with a custom one |
| `@Ignore`                                                     | Feature<br/>Meta<br/>Scenario<br/>Rule<br/>Background<br/>Examples | Ignores and skips over a Gherkin block/node to avoid evaluation                                                                      |
