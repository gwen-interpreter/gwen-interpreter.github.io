The Gwen logo consists of a [Lorenz attractor](https://en.m.wikipedia.org/wiki/Lorenz_system) that looks like a butterfly. Gwen evaluates software applications by mapping [Gherkin](https://cucumber.io/docs/gherkin/reference/) steps to instructions that:

- Create context (_Givens_)
- Perform actions (_Whens_)
- And check outcomes (_Thens_)

The cumulative result is a chain of instructions that execute to model cause and effect. If a small change in the application or environment breaks the chain, then Gwen will observe a [butterfly effect](https://en.m.wikipedia.org/wiki/Butterfly_effect). The proposition being that Gwen can detect chaotic behaviour.
