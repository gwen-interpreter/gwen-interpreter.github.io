# Gwen 4

> Released date: TBA

## What's new?

### Launch processes

You can now configure one or many [processes](/docs/processes#process-settings), each with its own launch options and [settings](/docs/settings). This enables you to configure reusable launch configurations for each process in your project so you don't have to manage them manually. You simply give each process a name and define its configuration in a same named settings file. Then when you're ready to launch, you specify the name of the process you want to run with the [`-p|--process`]((/docs/launcher#gwen-launch-options-process)) option.

- See [processes](/docs/processes#process-settings) for more info.

### New settings

The following [settings](/docs/settings) have been added:

| Setting                                                                   | Description                                                                                                                           |
| :------------------------------------------------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------ |
| [`gwen.input.data.readOnly`](/docs/settings#gwen-input-data-readonly)     | Enables or disables read-only mode for all data sourced from input files. Has no effect in REPL mode where read-only is not enforced. |

### New implicits

The following [implicit values](/docs/implicit-values) have been added:

| Name                                                           | Description                                                   |
| :------------------------------------------------------------- | :------------------------------------------------------------ |
| [`gwen.process.name`](/docs/implicit-values#gwen-process-name) |  The name of the currently running process (blank if default) |

### New environment variables

The following [environment variables](/docs/launcher#environment-variables) have been added:

| Name                 | Accepts                                      | Affects CLI option                              |
|:-------------------- | -------------------------------------------- | :---------------------------------------------: |
| `GWEN_PROCESS`       | Name of [process](/docs/processes) to launch | [`-p, --process`](#gwen-launch-options-process) |

### New DSLs

Introduced `empty` literal that can be used interchangeably with `blank` in all [DSL steps](/docs/dsl#dsl-reference) where `blank` is accepted.

### Force open REPL option

New [--repl](/docs/launcher#gwen-launch-options-repl) command line option added to for open REPL even if [-b|--batch](/docs/launcher#gwen-launch-options-batch) is specified. This is handy for cases where you might have batch specified in a helper script but want to use the REPL.

## Whats changed?

### Changed runtime requirements

- [Java](https://www.oracle.com/java/technologies/downloads/) 17+ is now the minimum supported Java version
- [Node](https://nodejs.org/en/download/package-manager) 18+ is now the minimum supported Node version

### Changed settings

Deprecated features are now raised as errors by default instead of being logged as warnings.

  | Setting                                                                  | Old default | New default |
  | :----------------------------------------------------------------------- | :---------: | :---------: |
  | [`gwen.logLevel.deprecations`](/docs/settings#gwen-loglevel-deprecations)|   `warn`    |   `error`   |

### Changed project structure

The `env` and `browsers` folders have been moved one level down under a `conf` subfolder which now also contains a `process` subfolder where you can define [process level settings](/docs/processes#process-level-settings).

The new project strucutre looks like this:

```
  ./                              # Project root
   ├── README.md
   ├── gwen.conf                  # Common settings
   └── /gwen
       ├── .gitignore             # Git ignore file
       ├── /conf
       │   ├── /browsers          # Browser settings
       │   |   ├── chrome.conf
       │   |   ├── edge.conf
       │   |   ├── firefox.conf
       │   |   ├── README.md
       │   |   └── safari.conf
       │   ├──/env                # Environment settings
       │   |  ├── dev.conf
       │   |  ├── local.conf
       │   |  ├── prod.conf
       │   |  ├── README.md
       │   |  ├── staging.conf
       │   |  └── test.conf
       │   └──/process            # Process settings
       │      ├── README.md
       │      ├── process1.conf
       │      ├── process3.conf
       │      ├── ..
       │      └── processN.conf
       ├── /features              # Features (and associative meta)
       │   └── README.md
       └── /meta                  # Common meta
           └── README.md
```

### Input data now read-only

All [input data](/docs/data-feeds) is now read-only at evaluation time by default (except in [REPL](/docs/repl) mode). This can be disabled through the [`gwen.input.data.readOnly`](/docs/settings#gwen-input-data-readonly) setting if required.

### Chaged logging implementation

Gwen now uses [Logback](https://logback.qos.ch/index.html) instead of Log4j for logging . You can override the default logging configuration with your by specifing `-Dlogback.configurationFile=/path-to-logback-config-file` of the command line or by setting `logback.configurationFile` in your settings.

## What's dropped?

### Dropped integrations
- IE browser integration
- Report Portal integration
- Standalone setup (only [project setup](/docs/get-started#setup) now supported)

### Dropped settings

The following [settings](/docs/settings) have been dropped. Update your [settings files](/docs/settings#settings-files) to use the replacement name.

  | Dropped                           | Use this instead                                                                                |
  | :-------------------------------- | :---------------------------------------------------------------------------------------------- |
  | `gwen.feature.failfast`           | [`gwen.feature.failfast.enabled`](/docs/settings#gwen-feature-failfast-enabled)                 |
  | `gwen-web-accept-untrusted-certs` | Use <code>acceptInsecureCerts</code> [capability](/docs/settings#gwen-web-capabilities) instead |
