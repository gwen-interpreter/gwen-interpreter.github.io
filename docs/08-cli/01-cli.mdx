# Gwen CLI

## Command line interface

:::tip

[Set up Gwen](/docs/get-started) to access the CLI.

:::

### Options and usage

All launch operations can be invoked using the Gwen CLI. To list the usage and all available options, invoke Gwen with only the [`--help`](#gwen-cli-options-help) switch. The following output will be displayed:

```
Usage: gwen [options] [files|dirs]

  --version               Print Gwen version
  --help                  Print CLI usage
  --parallel              Execute features or scenarios in parallel
                          - depending on gwen.state.level (default is feature)
  --parallel-features     Execute features in parallel (unconditionally)
  -b, --batch             Exit when execution completes (omit to open REPL)
  -v, --verbose           Enable verbose log output
  -n, --dry-run           Check all syntax and bindings and report errors
  -c, --conf files        Settings files: conf, json or properties (comma separated)
  -p, --properties files  Properties files (deprecated, use -c|--conf instead)
  -r, --report dir        Directory to output generated report(s) to
  -f, --formats reports   Report formats to include in output (comma separated)
                          - html,junit,json,rp,none (default is html)
  -t, --tags filter       Tags to @include or ~@exclude (comma separated)
  -i, --input-data file   Input data feed (CSV file with column headers)
  -m, --meta files|dirs   Meta files or directories to load (comma separated)
  files|dirs              Feature files or directories to execute (space separated)
Command: init [dir]
  dir                     Project directory to initialise (default is gwen)
```

### System properties and JVM options

In addition to the above, you can also pass any number of Java system properties or JVM options to the CLI:

- `-Dprop.name=value`
  - To pass a system property to the Java process that runs Gwen
  - Will override any same named [Gwen setting](/docs/settings/reference)
  - Will be picked up by `${prop.name}` placeholders in feature and meta specs
- `-J<JVM_OPTION>`
  - To pass a JVM option to the Java process that runs Gwen
  - The `-J` prefix will be stripped and `<JVM_OPTION>` will be passed through

### Configurable CLI options

As of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.0.0">v3.0.0</Link>, CLI options can be configured in [settings](/docs/settings/reference#cli-settings) enabling you to customize your own launch profile (excluding `--version` and `--help`).

### Init command

<details id="gwen-cli-options-init-dir">
  <summary class="cli-options">

  ```shell
  init [dir]
  ```
  <p>Command for initialising Gwen in a project directory.</p>

  </summary>

  <p>Where</p>

  <ul>
    <li><code>[dir]</code> is
      <ul>
        <li>Not required for JS projects
          <ul>
            <li>Defaults to a <code>gwen</code> subdirectory in your project</li>
          </ul>
        </li>
        <li>Must be specified as <code>.</code> for standalone projects (dot = current directory)</li>
      </ul>
    </li>
  </ul>

  <br />
  <p>
  Run this command in your project directory to initialise Gwen in it. You can then start using Gwen in your project.
  </p>
  <br />

  <p>Example</p>

  <Tabs
  groupId="setup"
  defaultValue="js"
  values={[
    {label: 'JS Project', value: 'js'},
    {label: 'Standalone', value: 'standalone'}
  ]}>

  <TabItem value="js">

  <Tabs
  groupId="jstool"
  defaultValue="npm"
  values={[
    {label: 'npm', value: 'npm'},
    {label: 'Yarn', value: 'yarn'},
    {label: 'pnpm', value: 'pnpm'}
  ]}>

  <TabItem value="npm">

  To initialse Gwen in a JS project
  
  ```shell
  npm run gwen init
  ```

  </TabItem>

  <TabItem value="yarn">

  To initialse Gwen in a JS project
  
  ```shell
  yarn gwen init
  ```

  </TabItem>

  <TabItem value="pnpm">

  To initialse Gwen in a JS project
  
  ```shell
  pnpm gwen init
  ```

  </TabItem>

  </Tabs>

  <p>The <code>init</code> command will generate the following Gwen files and subdirectories in your project</p>

  ```
    /js-project                 # Your JS project root
     gwen.conf               # Project settings
     /gwen
         .gitignore          # Git ignore file
         README.md
         /browsers           # Browser settings
            chrome.conf
            edge.conf
            firefox.conf
            safari.conf
            ie.conf
            remote.conf     # Remote web driver settings
            README.md
         /env                # Environment settings
            local.conf
            dev.conf
            test.conf
            README.md
         /features           # Features and associative meta
            README.md
         /meta               # Optional common/reusable meta
            README.md
         /samples            # Sample features and meta
  ```

  </TabItem>  

  <TabItem value="standalone">

  ```shell

  ```
