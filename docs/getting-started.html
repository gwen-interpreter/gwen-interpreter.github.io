Getting Started

Gwen Web#

Gwen-Web is a Gherkin interpreter that executes feature specs by binding them to meta specs that you write to drive automated web testing or repetitive online processesing using a Gherkin web DSL.

System Requirements#

  • Linux, Mac or Windows OS
  • Chrome, Firefox, Safari, Edge or IE web browser
  • Java JRE 8+ (version 1.8 or higher)
    • You can check if you have Java installed by opening a command prompt and typing java -version. A version will be displayed if you do have it installed (install Java).
  • node
    • Required if you intend to install and run Gwen from npm (install node)

Installing Gwen#

Install Gwen using one of the following methods:

  • npm
  • Yarn
  • pnpm
  • Standalone
npm i --save-dev @gweninterpreter/gwen-web
yarn add -D @gweninterpreter/gwen-web
pnpm add -D @gweninterpreter/gwen-web

Download a Gwen release ZIP, extract it to a desired location and add that to your system PATH

Using Gwen#

Once you have installed Gwen and initialised a working directory, achieving automation is a three step process:

1. Declare Features#

Write feature specs to describe the intented behaviour in a declarative manner without giving any thought to automation.

For example, create a new todo.feature file with the following content in your gwen/features folder.

File: gwen/features/todo.feature

Feature: Todo   Scenario: Create todo list    Given a new todo list     When the following items are added          | New Item      |          | Get the milk  |          | Walk the dog  |     Then the list will contain 2 items

2. Define Meta#

Write associated meta specs to define the step definitions for all steps in your features.

Each step definition is defined as a @StepDef annotated scenario with a name that matches the calling step and contains one or more steps that call out to other step definitions or predefined Gwen Web DSL steps that perform the prescribed browser operations. Steps containing Data tables can also be mapped to step definitions using the @ForEach and @DataTable annotations as shown in the example. Inputs to steps can also be parameterised using the <param> syntax and resolved with $<param>.

For example, create a new todo.meta file with the following content in your gwen/features folder.

Associative Meta: By default, Gwen will associate and bind same named *.meta files to same named *.feature files only when it discovers such pairs in the same directory!

File: gwen/features/todo.meta

Feature: Todo Meta (automation glue)
  @StepDef  Scenario: a new todo list       When I navigate to "http://todomvc.com/examples/react"       Then the todo field can be located by class "new-todo"        And count can be located by css ".todo-count strong"                          @StepDef  @ForEach  @DataTable(header="top")  Scenario: the following items are added       When I enter data[New Item] in the todo field
  @StepDef  Scenario: the list will contain <expected> items       Then count should be "$<expected>"

3. Launch Gwen#

Launch Gwen in your project root directory to execute your features!

  • npm
  • Yarn
  • pnpm
  • Standalone

Execute all features

npm run gwen -b /gwen/features

Execute one feature

npm run gwen -b /gwen/features/todo.feature

Execute all features

yarn gwen -b /gwen/features

Execute one feature

yarn gwen -b /gwen/features/todo.feature

Execute all features

pnpm gwen -b /gwen/features

Execute one feature

pnpm gwen -b /gwen/features/todo.feature

Execute all features

gwen -b /features

Execute one feature

gwen -b /features/todo.feature

The -b (--batch) switch tells Gwen to exit immediately after feature execution completes. Omitting it will open the interactive Gwen REPL Console upon completion instead.
