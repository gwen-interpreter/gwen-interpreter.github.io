# Launching Gwen

> Since [v1.0.0](https://github.com/gwen-interpreter/gwen-web/releases/tag/v1.0.0)

## Options and usage

All launch operations can be invoked using the Gwen CLI. To list the usage and all available options, invoke Gwen with only the [`--help`](#gwen-launch-options-help) option. The following output will be displayed:

```
Usage: gwen [options] [files|dirs]

  --version                Print Gwen version
  --help                   Print CLI usage
  --parallel               Execute features or scenarios in parallel
                           - features if gwen.state.level = feature (default)
                           - scenarios if gwen.state.level = scenario
  --repl                   Open REPL (forcefully) regardless of other options
  -p, --profile name       Name of profile to launch
  -b, --batch              Exit when execution completes (ignored if --repl is specified)
  -v, --verbose            Enable verbose log output
  -d, --debug              Enable breakpoints and interactive debugging
  -n, --dry-run            Check all syntax and bindings and report errors
  -c, --conf files         Settings files: conf, json or properties (comma separated)
  -r, --report dir         Directory to output generated report(s) to
  -f, --formats reports    Report formats to include in output (comma separated)
                           - html,results,junit,json,rp,none (default = html, rp deprecated)
  -t, --tags filter        Tags to @include or ~@exclude (comma separated)
  -i, --input-data file    Input data feed (csv or json file)
  -m, --meta files|dirs    Meta files or directories to load (comma separated)
  files|dirs               Feature files or directories to execute (space separated)
Command: init [dir]
Initialise project directory
  dir                      Project directory to initialise (default is gwen)
  --docker                 Generate files in project for running Gwen in Docker
  --jenkins                Generate files in project for running Gwen on Jenkins
  --force                  Regenerate and overwrite previously initialised files
Command: format files|dirs
Format feature and meta files
  files|dirs               Feature/meta files or dirs to format (space separated)
  --pretty                 Pretty Gherkin format (default)
```

### Environment variables

The following environment variables can be set to override certain settings as below:

| Since   | Environment variable | Values                                                                                                    | Overrides setting                                                             | Affects CLI option                              |
| :-----: |:-------------------- | --------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------- | :---------------------------------------------- |
| v3.23.0 | `GWEN_ENV`           | <code>local&#124;dev&#124;</code><br/><code>test&#124;staging&#124;</code><br/><code>prod&#124;..</code>  | [`gwen.target.env`](/docs/settings#gwen-target-env)                           |                                                 |
| v4.0.0  | `GWEN_PROFILE`       | Name of the [profile](/docs/profiles) to launch                                                              |                                                                            | [`-p, --profile`](#gwen-launch-options-profile) |
| v3.16.0 | `GWEN_BROWSER`       | <code>chrome&#124;edge&#124;</code><br/><code>firefox&#124;safari&#124;ie</code>                          | [`gwen.target.browser`](/docs/settings#gwen-target-browser)                   |                                                 |
| v3.16.0 | `GWEN_DRY_RUN`       | <code>true&#124;false</code>                                                                              | [`gwen.launch.options.dryRun`](/docs/settings#gwen-launch-options-dry-run)    | [`-n, --dry-run`](#gwen-launch-options-dry-run) |

### System properties and JVM options

In addition to the above, you can also pass any number of Java system properties or JVM options to the CLI:

- `-Dprop.name=value`
  - To pass a system property to the Java process that runs Gwen
  - Will override any same named [Gwen setting](/docs/settings)
  - Will be picked up by `${prop.name}` placeholders in feature and meta specs
- `-J<JVM_OPTION>`
  - To pass a JVM option to the Java process that runs Gwen
  - The `-J` prefix will be stripped and `<JVM_OPTION>` will be passed through

The enviornment variable overrides in the previous section however do have precedence over these.

<details id="gwen-launch-options-profile">
  <summary class="cli-options">

  ```shell
  -p|--profile <name>
  ```
  <p>
  Name of profile to launch.
  </p>

  </summary>

  <p>Where</p>

  <ul>
    <li>
      You can use either:
      <ul>
        <li><code>-p</code> for shorthand</li>
        <li><code>--profile</code> for longhand</li>
      </ul>
    </li>
    <li><code>&lt;name&gt;</code> is the name of a [profile](/docs/profiles) with associated <code>gwen/conf/profiles/&lt;name&gt;.conf</code> settings to load and launch with.</li>
  </ul>

  <p>Examples</p>

  To load all settings and execute all features defined in the <code>gwen/conf/profiles/custom.conf</code> settings and open [REPL](/docs/repl):
  
  ```shell
  gwen -p custom
  ```

  To load all settings and execute all features defined in the <code>gwen/conf/profiles/custom.conf</code> settings and exit when done:
  
  ```shell
  gwen -p custom -b
  ```

## Project Init

<details id="gwen-launch-init">
  <summary class="cli-options">

  ```shell
  init [options] [dir]
  ```
  <p>Command for initialising Gwen in a project directory.</p>

  </summary>

  <p>Where</p>

  <ul>
    <li>
      <code>[options]</code> Can be one or more space separated switches including:
      <ul>
        <li>
          <Link to="#gwen-launch-init-options-docker"><code>--docker</code></Link>
          <ul>
            <li>Include to also generate files in your project for running Gwen in <Link to="/docs/docker">Docker</Link></li>
          </ul>
        </li>
        <li>
          <Link to="#gwen-launch-init-options-jenkins"><code>--jenkins</code></Link>
          <ul>
            <li>Include to also generate a Jenkins pipeline file in your project</li>
          </ul>
        </li>
        <li>
          <code>--force</code>
          <ul>
            <li>Include to regenerate and overwrite any previously initialised files.</li>
          </ul>
        </li>
      </ul>
    </li>
    <li>
      <code>[dir]</code> is
      <ul>
        <li>Defaults to a <code>gwen</code> subdirectory in your project if not specified</li>
      </ul>
    </li>
  </ul>

  <br />
  <p>
  Run this command in your project directory to initialise Gwen in it. You can then start using Gwen in your project.
  </p>
  <br />

  <p>Example</p>

  <Tabs
  groupId="jstool"
  defaultValue="yarn"
  values={[
    {label: 'Yarn', value: 'yarn'},
    {label: 'npm', value: 'npm'},
    {label: 'pnpm', value: 'pnpm'}
  ]}>

  <TabItem value="yarn">

  To initialse a Gwen project
  
  ```shell
  yarn gwen init
  ```

  </TabItem>

  <TabItem value="npm">

  To initialse a Gwen project
  
  ```shell
  npm run gwen init
  ```

  </TabItem>

  <TabItem value="pnpm">

  To initialse a Gwen project
  
  ```shell
  pnpm gwen init
  ```

  </TabItem>

  </Tabs>

  <p>The <code>init</code> command will generate the following Gwen files and subdirectories in your project</p>

  ```
  ./                              # Project root
   ├── .gitignore                 # Git ignore file
   ├── README.md
   ├── gwen.conf                  # Common settings
   └── /gwen
       ├── .gitignore             # Git ignore file
       ├── /conf
       │   ├── /browsers          # Browser settings
       │   │   ├── chrome.conf
       │   │   ├── edge.conf
       │   │   ├── firefox.conf
       │   │   ├── README.md
       │   │   └── safari.conf
       │   ├── /env               # Environment settings
       │   │   ├── dev.conf
       │   │   ├── local.conf
       │   │   ├── prod.conf
       │   │   ├── README.md
       │   │   ├── staging.conf
       │   │   └── test.conf
       │   └── /profiles          # Profile settings
       │       ├── README.md
       │       └── samples.conf
       ├── /features              # Features and associative meta
       │   ├── README.md
       │   └── /samples          # Samples
       └── /meta                  # Common/reusable meta
           └── README.md
  ```

## Pretty formatting

<details id="gwen-launch-format-pretty">
  <summary class="cli-options">

  ```
  format --pretty <files|dirs>
  ```

  <ul>
    <li>
      <Link to="#gwen-launch-format-pretty"><code>--pretty</code></Link>
