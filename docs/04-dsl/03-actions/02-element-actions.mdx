<details id="i-click-element">
  <summary class="dsl">

  ```gherkin
  I <click|right click|double click> <element>
  ```
  <p>Clicks a web element.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;click|right click|double click&gt;</code> is one of:
      <ul>
        <li><code>click</code> to click the element</li>
        <li><code>right click</code> to right click the element, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.8.0">v2.8.0</Link></li>
        <li><code>double click</code> to double click the element, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.12.0">v2.12.0</Link></li>
      </ul>
    </li>
    <li><code>&lt;element&gt;</code> is the name of the element to click</li>
  </ul>
  <p>Example</p>

  ```gherkin {3}
    Given the start button can be located by name "commit"
     When I navigate to "https://challengers.flood.io/start"
      And I click the start button
  ```

    <div align="right">Since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v1.0.0">v1.0.0</Link></div>
  </div>
</details>

<details id="i-click-element-of-context">
  <summary class="dsl">

  ```gherkin
  I <click|right click|double click> <element> of <context>
  ```
  <p>Clicks a context sensitive web element.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;click|right click|double click&gt;</code> is one of:
      <ul>
        <li><code>click</code> to click the element</li>
        <li><code>right click</code> to right click the element, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.8.0">v2.8.0</Link></li>
        <li><code>double click</code> to double click the element, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.12.0">v2.12.0</Link></li>
      </ul>
    </li>
    <li><code>&lt;element&gt;</code> is the name of the element to click</li>
    <li><code>&lt;context&gt;</code> is the context element to focus into (in order to click <code>&lt;element&gt;</code>)</li>
  </ul>
  <p>Example</p>

  ```gherkin {3}
    Given the menu item can be located by id "menu-item-id"
      And the action button can be located by id "action-button-id"
     When I click the action button of the menu item
  ```

<details id="i-modifier-keys-click-element">
  <summary class="dsl">

  ```gherkin
  I <modifierKeys> <click|right click|double click> <element>
  ```
  <p>Holds down the specified modifier keys and clicks an element.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;modifierKeys&gt;</code> is a <code>+</code> separated list of modifier keys to hold down during the click
      <ul>
        <li>Modifier keys can be single keyboard characters or any supported <Link to="https://seleniumhq.github.io/selenium/docs/api/java/org/openqa/selenium/Keys.html">key constants</Link></li>
        <li><i>Node:</i> Keys behave differently on different platforms</li>
      </ul>
    </li>
    <li>
      <code>&lt;click|right click|double click&gt;</code> is one of:
      <ul>
        <li><code>click</code> to click the element</li>
        <li><code>right click</code> to right click the element</li>
        <li><code>double click</code> to double click the element, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.12.0">v2.12.0</Link></li>
      </ul>
    </li>
    <li><code>&lt;element&gt;</code> is the name of the element to click</li>
  </ul>
  <p>Example</p>

  ```gherkin {2}
    Given the button can be located by tag "button"
      And I COMMAND+SHIFT click the button
  ```

  </ul>
  <p>Example</p>


<details id="i-enter-or-type-text-in-element">
  <summary class="dsl">

  ```gherkin
  I <enter|type> "<text>" in <element>
  ```
  <p>Enters or types text into a web element.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;enter|type&gt;</code> is one of:
      <ul>
        <li><code>enter</code> to send the text to the field followed by the enter key</li>
        <li><code>type</code> to send the text to the field only (without trailing enter key)</li>
      </ul>
    </li>
    <li><code>&lt;text&gt;</code> is the text to send to the field</li>
    <li><code>&lt;element&gt;</code> is the name of the field to send the text to</li>
  </ul>
  <p>Example</p>

  ```gherkin {3}
    Given the todo field can be located by class "new-todo"
     When I navigate to "https://todomvc.com/examples/react/dist"
      And I enter "Walk the dog" in the todo field
  ```

    <div align="center"></div>
    <div align="right">Since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v1.0.0">v1.0.0</Link></div>

<details id="i-enter-or-type-text-ref-in-element">
  <summary class="dsl">

  ```gherkin
  I <enter|type> <textRef> in <element>
  ```
  <p>Enters or types a text reference value into a web element.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;enter|type&gt;</code> is one of:
      <ul>
        <li><code>enter</code> to send the text to the field followed by the enter key</li>
        <li><code>type</code> to send the text to the field only (without trailing enter key)</li>
      </ul>
    </li>
    <li>
      <code>&lt;textRef&gt;</code> is the name of the binding containing the text to send to the field
      <ul>
        <li>Can be the name of any binding that contains or resolves to a text value, including a web element in which case the text in the element will be dynamically retrieved and used.</li>
      </ul>
    </li>
    <li><code>&lt;element&gt;</code> is the name of the field to send the text to</li>
  </ul>
  <p>Example</p>

  ```gherkin {4}
    Given the todo field can be located by class "new-todo"
      And my item is "Walk the dog"
     When I navigate to "https://todomvc.com/examples/react/dist"
      And I enter my item in the todo field
  ```

<details id="i-send-keys-to-element">
  <summary class="dsl">

  ```gherkin
  I send "<keys>" to <element>
  ```
  <p>Sends a sequence of keys to an element (for emulating keyboard shortcuts).</p>
  </summary>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;keys&gt;</code> is a <code>,</code> or <code>+</code> (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.53.0">v2.53.0</Link>) separated list of keys to send.
      <ul>
        <li>Keys can be single keyboard characters or any supported <Link to="https://seleniumhq.github.io/selenium/docs/api/java/org/openqa/selenium/Keys.html">key constants</Link></li>
        <li><i>Note:</i> Keys behave differently on different platforms</li>
      </ul>
    </li>
    <li><code>&lt;element&gt;</code> is the name of the element to send the keys to</li>
  </ul>
  <p>Example</p>

  ```gherkin {4,5,7,8}
    Given the search field can be located by name "q"
     When I navigate to "https://google.com"
      And I type "automation" in the search field
      And I send "COMMAND+A" to the search field
      And I send "COMMAND+C" to the search field
      And I clear the search field
      And I send "COMMAND+V" to the search field
      And I send "ARROW_DOWN" to the search field
      And I press enter in the search field
  ```
