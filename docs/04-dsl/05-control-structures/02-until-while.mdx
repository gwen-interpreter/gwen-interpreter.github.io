<details id="step-until-or-while-condition">
  <summary class="dsl">

  ```gherkin
  <step> <until|while> <condition>
  ```
  <p>Repeatedly exectues a step until or while a condition is satisfied using a default delay and timeout period. The default delay between iterations is one tenth of the configured <code>gwen.web.wait.seconds</code> setting (or 1 second if not set). The default timeout period is 30 times the delay. The default delay and timeout can be overridden using the `@Delay` and `@Timeout` annotations.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;step&gt;</code> is the DSL step or StepDef to execute in each iteration</li>
    <li>
      <code>&lt;until|while&gt;</code> is one of:
      <ul>
        <li><code>until</code> to repeat until the condition is satisfied</li>
        <li><code>while</code> to repeat while the condition is satisfied</li>
      </ul>
    </li>
    <li>
      <code>&lt;condition&gt;</code> is either:
      <ul>
        <li>
          Since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.54.0">v3.54.0</Link>
          <ul>
            <li>the name of any binding that will return <code>true</code> when the condition is satisfied or <code>false</code> when it is not</li>
          </ul>
        </li>
        <li>
          Or in prior versions
          <ul>
            <li>the name of a binding containing a javascript function or expression that will return <code>true</code> when the condition is satisfied or <code>false</code> when it is not</li>
            <li>the name of a binding containing a <code>true</code> or <code>false</code> value literal, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.50.0">v3.50.0</Link></li>
          </ul>
        </li>
      </ul>
    </li>
    <li>
      The following implicit binding is available in <code>&lt;step&gt;</code>:
      <ul>
        <li><code>iteration number</code> (or <code>gwen.iteration.number</code> since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.52.0">v2.52.0</Link>) is the current iteration number (starting at 1)</li>
      </ul>
    </li>
  </ul>
  <p>Example</p>

  ```gherkin {13}
      Given the search field can be located by name "q"
        And the next page link can be located by id "pnnext"
        And topic is "what is gwen"
        And topic link can be located by partial link text "${topic}"
        And topic not found is defined by js
            """
            [...document.querySelectorAll("a")]
              .filter(a => a.textContent.includes('${topic}'))
              .length == 0
            """
       When I navigate to "https://google.com"
        And I enter "gwen interpreter" in the search field
        And I click the next page link while topic not found
        And I click topic link
       Then the page title should contain "${topic}"
  ```

  With explicit delay and timeout since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>

  ```gherkin {13}
      Given the search field can be located by name "q"
        And the next page link can be located by id "pnnext"
        And topic is "what is gwen"
        And topic link can be located by partial link text "${topic}"
        And topic not found is defined by js
            """
            [...document.querySelectorAll("a")]
              .filter(a => a.textContent.includes('${topic}'))
              .length == 0
            """
       When I navigate to "https://google.com"
        And I enter "gwen interpreter" in the search field
        And @Delay('10s') @Timeout('1m') I click the next page link while topic not found
        And I click topic link
       Then the page title should contain "${topic}"
  ```

  With no delay or timeout since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>

  ```gherkin {13}
      Given the search field can be located by name "q"
        And the next page link can be located by id "pnnext"
        And topic is "what is gwen"
        And topic link can be located by partial link text "${topic}"
        And topic not found is defined by js
            """
  ```

<details id="step-until-or-while-element-state">
  <summary class="dsl">

  ```gherkin
  <step> <until|while> <element> is[ not] <state>
  ```
  <p>Repeatedly exectues a step until or while an element is in a given state using a 1 second delay interval and 1 minute timeout period. The default delay and timeout can be overridden using the `@Delay` and `@Timeout` annotations.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;step&gt;</code> is the DSL step or StepDef to execute in each iteration</li>
    <li>
      <code>&lt;until|while&gt;</code> is one of:
      <ul>
        <li><code>until</code> to repeat until the condition is satisfied</li>
        <li><code>while</code> to repeat while the condition is satisfied</li>
      </ul>
    </li>
    <li><code>&lt;element&gt;</code> is the name of the element to check the state of</li>
    <li><code>not</code> negates the state if included</li>
    <li>
      <code>&lt;state&gt;</code> is one of:
      <ul>
        <li><code>displayed</code> to execute the step until or while the element is displayed</li>
        <li><code>hidden</code> to execute the step until or while the element is not displayed</li>
        <li><code>checked</code> or <code>ticked</code> to execute the step until or while a checkbox is ticked</li>
        <li><code>unchecked</code> or <code>unticked</code> to execute the step until or while a checkbox is not ticked</li>
        <li><code>enabled</code> to execute the step until or while the element is enabled</li>
        <li><code>disabled</code> to execute the step until or while the element is disabled</li>
      </ul>
    </li>
    <li>
      The following implicit binding is available in <code>&lt;step&gt;</code>:
      <ul>
        <li><code>iteration number</code> (or <code>gwen.iteration.number</code> since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.52.0">v2.52.0</Link>) is the current iteration number (starting at 1)</li>
      </ul>
    </li>
  </ul>
  <p>Example</p>

  ```gherkin {5}
      Given the search field can be located by name "q"
        And the next page link can be located by id "pnnext"
       When I navigate to "https://google.com"
        And I enter "gwen interpreter" in the search field
        And I click the next page link while the next page link is displayed
       Then the next page link should not be displayed
  ```

  With explicit delay and timeout since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>

   ```gherkin {5}
      Given the search field can be located by name "q"
        And the next page link can be located by id "pnnext"
       When I navigate to "https://google.com"
        And I enter "gwen interpreter" in the search field
        And @Delay('5s') @Timeout('40s') I click the next page link while the next page link is displayed
       Then the next page link should not be displayed
  ```

  With no delay or timeout since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>

   ```gherkin {5}
      Given the search field can be located by name "q"
        And the next page link can be located by id "pnnext"
       When I navigate to "https://google.com"
        And I enter "gwen interpreter" in the search field
        And @Delay('0s') @Timeout('0s') I click the next page link while the next page link is displayed
       Then the next page link should not be displayed
  ```

