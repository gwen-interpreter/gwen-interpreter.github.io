<details id="step-if-condition">
  <summary class="dsl">

  ```gherkin
  <step> if[ not] <condition>
  ```
  <p>Executes a step depending on whether or not a JS condition evaluates to true.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;step&gt;</code> is the DSL step or StepDef to potentially execute</li>
    <li><code>not</code> negates the condition if included (Since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.4.0">v3.4.0</Link>)</li>
    <li>
      <code>&lt;condition&gt;</code> is either:
      <ul>
        <li>
          Since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.54.0">v3.54.0</Link>
          <ul>
            <li>the name of any binding that will return <code>true</code> when the condition is satisfied or <code>false</code> when it is not</li>
          </ul>
        </li>
        <li>
          Or in prior versions
          <ul>
            <li>the name of a binding containing a javascript function or expression that will return <code>true</code> when the condition is satisfied or <code>false</code> when it is not</li>
            <li>the name of a binding containing a <code>true</code> or <code>false</code> value literal, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.50.0">v3.50.0</Link></li>
          </ul>
        </li>
        <li>Since v3.14.1, the condition name cannot contain double qutoes</li>
      </ul>
    </li>
  </ul>
  <p>Example</p>

  ```gherkin {12,13}
    Given the search field can be located by name "q"
      And topic is "Automation Bindings"
      And topic link can be located by partial link text "${topic}"
      And topic found is defined by js
          """
          [...document.querySelectorAll("a")]
              .filter(a => a.textContent.includes('${topic}'))
              .length > 0
          """
     When I navigate to "https://google.com"
      And I enter "automation" in the search field
      And I click topic link if topic found
     Then the page title should contain "${topic}" if topic found
  ```

<details id="step-if-element-condition">
  <summary class="dsl">

  ```gherkin
  <step> if <element> is[ not] <state>
  ```
  <p>Executes a step depending on whether or not an element is in a given state.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;step&gt;</code> is the DSL step or StepDef to potentially execute</li>
    <li><code>&lt;element&gt;</code> is the name of the element to check the state of</li>
    <li><code>not</code> negates the state if included</li>
    <li>
      <code>&lt;state&gt;</code> is one of:
      <ul>
        <li><code>displayed</code> to execute the step if the element is displayed</li>
        <li><code>hidden</code> to execute the step if the element is not displayed</li>
        <li><code>checked</code> or <code>ticked</code> to execute the step if a checkbox is ticked</li>
        <li><code>unchecked</code> or <code>unticked</code> to execute the step if a checkbox is not ticked</li>
        <li><code>enabled</code> to execute the step if the element is enabled</li>
        <li><code>disabled</code> to execute the step if the element is disabled</li>
      </ul>
    </li>
  </ul>
  <p>Example</p>

  ```gherkin {6}
    Given the search field can be located by name "q"
      And topic is "Automation"
      And topic link can be located by partial link text "${topic}"
     When I navigate to "https://google.com"
      And I enter "automation" in the search field
      And I click topic link if topic link is displayed
     Then the page title should contain "${topic}"
  ```

<details id="step-if-binding-defined">
  <summary class="dsl">

  ```gherkin
  <step> if <name> is[ not] defined
  ```
  <p>Executes a step depending on whether or not an attribute is bound in the global or current scope.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;step&gt;</code> is the DSL step or StepDef to potentially execute</li>
    <li><code>&lt;name&gt;</code> is the name of the attribute (or StepDef &lt;param&gt; since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v4.11.0">v4.11.0</Link>) to check</li>
    <li><code>not</code> include to execute step if attribute is not bound</li>
  </ul>
  <p>Example</p>

  Given the CSV file: lookup/StateCodes.csv

  ```csv {3}
  Code,Name
  ACT,Australian Capital Territory
  NSW,New South Wales
  NT,Northern Territory
  QLD,Queensland
  SA,South Australia
  TAS,Tasmania
  VIC,Victoria
  WA,Western Australia
  ```

  The following will bind "New South Wales" in the 2nd record to <code>state name</code> in the global scope
  ```gherkin {3}
   Given the state code is "UNKNOWN"
    When I lookup Name in the "lookup/StateCodes.csv" file as state name where "'${data.record.Code}' == '${the state code}'"
    Then unresolved is "true" if state name is not defined
  ```

<details id="step-if-binding-is-value">
  <summary class="dsl">

  ```gherkin
  <step> if <name> is[ not] "<value>"
  ```
  <p>Executes a step depending on whether or not an element or attribute is equal to a value.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;step&gt;</code> is the DSL step or StepDef to potentially execute</li>
    <li><code>&lt;name&gt;</code> is the name of the element or attribute (or StepDef &lt;param&gt; since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v4.11.0">v4.11.0</Link>) to check</li>
    <li><code>not</code> include to execute step if the attribute is not equal to the given value</li>
    <li><code>value</code> is the given value</li>
  </ul>
  <p>Example</p>

  ```gherkin {2-3}
   Given the state code can be located by name "state"
    When I navigate to "https://example.com?state=Victoria" if the state code is "VIC"
     And I navigate to "https://example.com?state=Other" if the state code is not "VIC"
  ```

  Trimming and ignoring case (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
    <li>The <code>@IgnoreCase</code> annotation can be used on step to ignore case when comparing</li>
  </ul>

<details id="step-if-binding-match">
  <summary class="dsl">

  ```gherkin
  <step> if <name> <match> "<expression>"`
  ```
  <p>Executes a step depending on whether or not an element or attribute matches an expression.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;step&gt;</code> is the DSL step or StepDef to potentially execute</li>
    <li><code>&lt;name&gt;</code> is the name of the element or attribute (or StepDef &lt;param&gt; since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v4.11.0">v4.11.0</Link>) to check</li>
    <li>
      <code>&lt;match&gt;</code> is the type of match to perform, one of:
      <ul>
        <li><code>contains</code> for partial match</li>
        <li><code>starts with</code> for partial leading match</li>
        <li><code>ends with</code> for partial trailing match</li>
        <li><code>matches regex</code> for regex match</li>
        <li><code>matches xpath</code> for XML match</li>
        <li><code>matches json path</code> for <Link to="https://github.com/json-path/JsonPath/blob/master/README.md">JSON match</Link>, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v1.4.0">v1.4.0</Link></li>
        <li><code>matches template</code> for <Link to="/docs/template-matching">template</Link> match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
        <li><code>matches template file</code> for <Link to="/docs/template-matching">template</Link> file match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
      </ul>
    </li>
    <li><code>&lt;expression&gt;</code> is the expression to match (can be in <Link to="/docs/doc-strings">DocString</Link> position)</li>
  </ul>
  <p>Example</p>

  ```gherkin {2}
   Given the state code can be located by name "state"
    When I navigate to "https://example.com?state=${the state code}" if the state code matches regex "(VIC|NSW)"
  ```

  Trimming and ignoring case (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
    <li>The <code>@IgnoreCase</code> annotation can be used on step to ignore case when comparing</li>
  </ul>
