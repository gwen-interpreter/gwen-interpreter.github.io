<details id="text-ref-should-match-blank">
  <summary class="dsl">

  ```gherkin
  <textRef> should[ not] be <blank|empty>
  ```

  <p>Checks whether or not the text reference matches a blank or empty string. Reports an assertion error if the match fails.</p>
  </summary>
  <p>

  <p>With custom assertion error message (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.9.0">v3.9.0</Link>):</p>

  ```gherkin
  <textRef> should[ not] be <blank|empty>  @Message("my custom assert fail message")
  ```

  </p>
  <hr/>
  <p>
  <p>Without any timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('0s') <textRef> should[ not] be blank
  ```
  
  </p>
  <hr/>
  <p>
  <p>With custom timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('10s') <textRef> should[ not] be blank
  ```
  
  </p>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;textRef&gt;</code> is the name of the binding containing the text to check
      <ul>
        <li>Can be the name of any binding that contains or resolves to a text value, including a web element in which case the text in the element will be dynamically retrieved and used.</li>
      </ul>
    </li>
    <li><code>not</code> negates the match if included</li>
    <li><code>&lt;timeoutSecs&gt;</code> is the timeout period in seconds</li>
    <li><code>&lt;wait|timeout&gt;</code> is either <code>wait</code> or <code>timeout</code></li>
  </ul>
  <p>Example</p>

  ```gherkin {6}
    Given the todo field can be located by class "new-todo"
      And my todo list can be located by css ".todo-list"
     When I navigate to "https://todomvc.com/examples/react/dist"
      And I enter "Get the milk" in the todo field
      And I capture my todo list as current items
     Then current items should not be empty
  ```

  Trimming (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
  </ul>


  ```gherkin
  @Timeout('0s') <textRef> should[ not] <match> "<expression>"
  ```
  
  </p>
  <hr/>
  <p>
  <p>With custom timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('10s') <textRef> should[ not] <match> "<expression>"
  ```

  </p>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;textRef&gt;</code> is the name of the binding containing the text to check
      <ul>
        <li>Can be the name of any binding that contains or resolves to a text value, including a web element in which case the text in the element will be dynamically retrieved and used.</li>
      </ul>
    </li>
    <li><code>not</code> negates the match if included</li>
    <li>
      <code>&lt;match&gt;</code> is the type of match to perform, one of:
      <ul>
        <li><code>be</code> for exact match</li>
        <li><code>contain</code> for partial match</li>

        <li><code>end with</code> for partial trailing match</li>

        <li><code>match xpath</code> for XML match</li>

    </li>
    <li><code>&lt;expression&gt;</code> is the expression to match (can be in <Link to="/docs/doc-strings">DocString</Link> position)</li>
    <li><code>&lt;timeoutSecs&gt;</code> is the timeout period in seconds</li>
    <li><code>&lt;wait|timeout&gt;</code> is either <code>wait</code> or <code>timeout</code></li>

  ```gherkin {6}
      And I enter "Get the milk" in the todo field
      And I capture my todo list as current items
     Then current items should contain "Get the milk"
  ```

  Trimming and ignoring case (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
    <li>The <code>@IgnoreCase</code> annotation can be used on step to ignore case when comparing</li>
  </ul>

<details id="text-ref-should-be-similar-to-text">
  <summary class="dsl">

  ```gherkin
  <textRef> should[ not] <match> <percentage>% similar to "<text>"
  ```

  <p>Compares a text reference with a text value for similarity using the DSC (Dice Similarity Coefficient) algorithm.</p>
  </summary>
  <p>
  <p>With custom assertion error message (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.9.0">v3.9.0</Link>):</p>

  ```gherkin
  <textRef> should[ not] <match> <percentage>% similar to "<text>"  @Message("my custom assert fail message")
  ```

  </p>
  <hr/>
  <p>
  <p>Without any timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('0s') <textRef> should[ not] <match> <percentage>% similar to "<text>"
  ```

  </p>
  <hr/>
  <p>
  <p>With custom timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('10s') <textRef> should[ not] <match> <percentage>% similar to "<text>"
  ```

  </p>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;textRef&gt;</code> is the name of the binding containing a text reference
      <ul>
        <li>Can be the name of any binding that contains or resolves to a text value, including a web element in which case the text in the element will be dynamically retrieved and used.</li>
      </ul>
    </li>
    <li><code>not</code> negates the match if included</li>
    <li>
      <code>match</code> is one of:
      <ul>
        <li><code>be</code> for exact match</li>
        <li><code>be less than</code> for less than match</li>
        <li><code>be at most</code> for less than or equal to match</li>
        <li><code>be more than</code> for greater than match</li>
        <li><code>be at least</code> for greater than or equal to match</li>
      </ul>
    </li>
    <li><code>&lt;text&gt;</code> is the text to compare <code>&lt;textRef&gt;</code> against</li>
    <li><code>&lt;timeoutSecs&gt;</code> is the timeout period in seconds</li>
    <li><code>&lt;wait|timeout&gt;</code> is either <code>wait</code> or <code>timeout</code></li>
  </ul>
  The calculated similarity score is implicity bound to the <code>similarity score</code> attribute in the global scope.
  <p>Examples</p>

  ```gherkin {2-7}
    Given the phrase is "Hello world!"
     Then the phrase should be more than 90% similar to "Hello world"
      And @IgnoreCase the phrase should be at least 90% similar to "hello world"
      And the phrase should be 94.12% similar to "Hello world"
      And @IgnoreCase the phrase should be 94.12% similar to "hello world"
      And the phrase should be 82.35% similar to "hello world"
      And the phrase should not be less than 90% similar to "Hello world"
  ```

  Trimming and ignoring case (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
    <li>The <code>@IgnoreCase</code> annotation can be used on step to ignore case when comparing</li>
  </ul>
