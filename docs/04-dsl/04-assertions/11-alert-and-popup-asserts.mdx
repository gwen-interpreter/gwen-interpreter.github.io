<details id="the-alert-or-confirmation-popup-message-should-match-blank">
  <summary class="dsl">

  ```gherkin
  the <alert|confirmation> popup message should[ not] be <blank|empty>
  ```
  
  <p>Checks whether or not the message in the alert or confirmation popup box matches a blank or empty string. Reports an assertion error if the match fails.</p>
  </summary>
  <p>
  <p>With custom assertion error message (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.9.0">v3.9.0</Link>):</p>

  ```gherkin
  the <alert|confirmation> popup message should[ not] be blank  @Message("my custom assert fail message")
  ```

  </p>
  <hr/>
  <p>
  <p>Without any timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('0s') the <alert|confirmation> popup message should[ not] be blank
  ```

  </p>
  <hr/>
  <p>
  <p>With custom timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('10s') the <alert|confirmation> popup message should[ not] be blank
  ```

  </p>
  <p>Where</p>
  <ul>
    <li><code>not</code> negates the match if included</li>
    <li><code>&lt;timeoutSecs&gt;</code> is the timeout period in seconds</li>
    <li><code>&lt;wait|timeout&gt;</code> is either <code>wait</code> or <code>timeout</code></li>
  </ul>
  <p>Example</p>

  ```gherkin {4}
    Given the alert link can be located by id "alert"
     When I navigate to "https://www.selenium.dev/selenium/web/alerts.html"
      And I click the alert link
     Then the alert popup message should not be blank
  ```

  Trimming (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
  </ul>

<details id="the-alert-or-confirmation-popup-message-should-match-expression">
  <summary class="dsl">

  ```gherkin
  the <alert|confirmation> popup message should[ not] <match> "<expression>"
  ```
  
  <p>Checks whether or not the message in the alert or confirmation popup box matches a given expression. Reports an assertion error if the match fails.</p>
  </summary>
  <p>
  <p>With custom assertion error message (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.9.0">v3.9.0</Link>):</p>

  ```gherkin
  the <alert|confirmation> popup message should[ not] <match> "<expression>"  @Message("my custom assert fail message")
  ```

  </p>
  <hr/>
  <p>
  <p>Without any timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('0s') the <alert|confirmation> popup message should[ not] <match> "<expression>"
  ```

  </p>
  <hr/>
  <p>
  <p>With custom timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('10s') the <alert|confirmation> popup message should[ not] <match> "<expression>"
  ```

  </p>
  <p>Where</p>
  <ul>
    <li><code>not</code> negates the match if included</li>
    <li>
      <code>&lt;match&gt;</code> is the type of match to perform, one of:
      <ul>
        <li><code>be</code> for exact match</li>
        <li><code>contain</code> for partial match</li>
        <li><code>start with</code> for partial leading match</li>
        <li><code>end with</code> for partial trailing match</li>
        <li><code>match regex</code> for regex match</li>
        <li><code>match xpath</code> for XML match</li>
        <li><code>match json path</code> for <Link to="https://github.com/json-path/JsonPath/blob/master/README.md">JSON match</Link>, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v1.4.0">v1.4.0</Link></li>
        <li><code>match template</code> for <Link to="/docs/template-matching">template</Link> match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
        <li><code>match template file</code> for <Link to="/docs/template-matching">template</Link> file match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
      </ul>
    </li>
    <li><code>&lt;expression&gt;</code> is the expression to match (can be in <Link to="/docs/doc-strings">DocString</Link> position)</li>
    <li><code>&lt;timeoutSecs&gt;</code> is the timeout period in seconds</li>
    <li><code>&lt;wait|timeout&gt;</code> is either <code>wait</code> or <code>timeout</code></li>
  </ul>
  <p>Example</p>

  ```gherkin {4}
    Given the alert link can be located by id "alert"
     When I navigate to "https://www.selenium.dev/selenium/web/alerts.html"
      And I click the alert link
     Then the alert popup message should be "cheese"
  ```

  Trimming and ignoring case (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
    <li>The <code>@IgnoreCase</code> annotation can be used on step to ignore case when comparing</li>
  </ul>

<details id="the-alert-or-confirmation-popup-message-should-match-expression-ref">
  <summary class="dsl">

  ```gherkin
  the <alert|confirmation> popup message should[ not] <match> <expressionRef>
  ```
  
  <p>Checks whether or not the message in the alert or confirmation popup box matches a referenced expression. Reports an assertion error if the match fails.</p>
  </summary>
  <p>
  <p>With custom assertion error message (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.9.0">v3.9.0</Link>):</p>

  ```gherkin
  the <alert|confirmation> popup message should[ not] <match> <expressionRef>  @Message("my custom assert fail message")
  ```

  </p>
  <hr/>
  <p>
  <p>Without any timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('0s') the <alert|confirmation> popup message should[ not] <match> <expressionRef>
  ```

  </p>
  <hr/>
  <p>
  <p>With custom timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('10s') the <alert|confirmation> popup message should[ not] <match> <expressionRef>
  ```

  </p>
  <p>Where</p>
  <ul>
    <li><code>not</code> negates the match if included</li>
    <li>
      <code>&lt;match&gt;</code> is the type of match to perform, one of:
      <ul>
        <li><code>be</code> for exact match</li>
        <li><code>contain</code> for partial match</li>
        <li><code>start with</code> for partial leading match</li>
        <li><code>end with</code> for partial trailing match</li>
        <li><code>match regex</code> for regex match</li>
        <li><code>match xpath</code> for XML match</li>
        <li><code>match json path</code> for <Link to="https://github.com/json-path/JsonPath/blob/master/README.md">JSON match</Link>, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v1.4.0">v1.4.0</Link></li>

        <li><code>match template file</code> for <Link to="/docs/template-matching">template</Link> file match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
      </ul>
    </li>
    <li>
      <code>&lt;expressionRef&gt;</code> is the name of the binding containing the expression to match
      <ul>
        <li>Can be the name of any binding that contains or resolves to an expression value, including a web element in which case the text in the element will be dynamically retrieved and used.</li>
      </ul>
    </li>
    <li><code>&lt;timeoutSecs&gt;</code> is the timeout period in seconds</li>
    <li><code>&lt;wait|timeout&gt;</code> is either <code>wait</code> or <code>timeout</code></li>
  </ul>
  <p>Example</p>

  ```gherkin {5}

      And the expected message is "Enter something"
     When I navigate to "https://www.selenium.dev/selenium/web/alerts.html"
      And I click the confirmation link
     Then the confirmation popup message should be the expected message
  ```

  Trimming and ignoring case (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>

<details id="the-alert-or-confirmation-popup-should-be-displayed">
  <summary class="dsl">

  ```gherkin
  the <alert|confirmation> popup should[ not] be displayed
  ```
  
  <p>Checks whether or not the alert/comnfirmation popup should be displayed or not. Reports an assertion error if the assertion fails.</p>
  </summary>
  <p>
  <p>With custom assertion error message:</p>

  ```gherkin
  the <alert|confirmation> popup should[ not] be displayed  @Message("my custom assert fail message")
  ```

  </p>
  <hr/>
  <p>
  <p>Without any timeout:</p>

  ```gherkin
  @Timeout('0s') the <alert|confirmation> popup should[ not] be displayed
  ```

  </p>
  <hr/>
  <p>
  <p>With custom timeout:</p>

  ```gherkin
  @Timeout('10s') the <alert|confirmation> popup should[ not] be displayed
  ```

  </p>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;alert|confirmation&gt;</code> is either:
      <ul>
        <li><code>alert</code> for an alert box with an OK button</li>
        <li><code>confirmation</code> for a confirmation box with OK and Cancel buttons</li>
      </ul>
    </li>
    <li><code>not</code> negates the assertion if included (should not be displayed)</li>
  </ul>
  <p>Example</p>

  ```gherkin {4}
    Given the confirmation link can be located by id "prompt-with-default"
     When I navigate to "https://www.selenium.dev/selenium/web/alerts.html"
      And I click the confirmation link
     Then the confirmation popup should be displayed
  ```
