<details id="drodown-text-or-value-should-match-blank">
  <summary class="dsl">

  ```gherkin
  <dropdown> <text|value> should[ not] be <blank|empty>
  ```
  
  <p>Checks whether or not a dropdown selection matches a blank or empty string. Reports an assertion error if the match fails.</p>
  </summary>
  <p>
  <p>With custom assertion error message (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.9.0">v3.9.0</Link>):</p>

  ```gherkin
  <dropdown> <text|value> should[ not] be blank  @Message("my custom assert fail message")
  ```

  </p>
  <hr/>
  <p>
  <p>Without any timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('0s') <dropdown> <text|value> should[ not] be blank
  ```

  </p>
  <hr/>
  <p>
  <p>With custom timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('10s') <dropdown> <text|value> should[ not] be blank
  ```

  </p>
  <p>Where</p>
  <ul>
    <li><code>&lt;dropdown&gt;</code> is the dropdown element to check</li>
    <li>
      <code>&lt;text|value&gt;</code> is one of:
      <ul>
        <li><code>text</code> to check the displayed text of the selection</li>
        <li><code>value</code> to check the option value of the selection</li>
      </ul>
    </li>
    <li><code>not</code> negates the match if included</li>
    <li><code>&lt;timeoutSecs&gt;</code> is the timeout period in seconds</li>
    <li><code>&lt;wait|timeout&gt;</code> is either <code>wait</code> or <code>timeout</code></li>
  </ul>
  <p>Example</p>

  ```gherkin {4}
    Given the age dropdown can be located by id "challenger_age"
     When I navigate to "https://challengers.flood.io/step/2"
      And I select the 2nd option in the age dropdown
     Then the age dropdown value should not be blank
  ```

  Trimming (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
  </ul>

<details id="drodown-text-or-value-should-match-expression">
  <summary class="dsl">

  ```gherkin
  <dropdown> <text|value> should[ not] <match> "<expression>"
  ```
  
  <p>Checks whether or not a dropdown selection matches a given expression. Reports an assertion error if the match fails.</p>
  </summary>
  <p>
  <p>With custom assertion error message (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.9.0">v3.9.0</Link>):</p>

  ```gherkin
  <dropdown> <text|value> should[ not] <match> "<expression>"  @Message("my custom assert fail message")
  ```

  </p>
  <hr/>
  <p>
  <p>Without any timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('0s') <dropdown> <text|value> should[ not] <match> "<expression>"
  ```

  </p>
  <hr/>
  <p>
  <p>With custom timeout (as of <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.73.0">v3.73.0</Link>):</p>

  ```gherkin
  @Timeout('10s') <dropdown> <text|value> should[ not] <match> "<expression>"
  ```

  </p>
  <p>Where</p>
  <ul>
    <li><code>&lt;dropdown&gt;</code> is the dropdown element to check</li>
    <li>
      <code>&lt;text|value&gt;</code> is one of:
      <ul>
        <li><code>text</code> to check the displayed text of the selection</li>
        <li><code>value</code> to check the option value of the selection</li>
      </ul>
    </li>
    <li><code>not</code> negates the match if included</li>
    <li>
      <code>&lt;match&gt;</code> is the type of match to perform, one of:
      <ul>
        <li><code>be</code> for exact match</li>
        <li><code>contain</code> for partial match</li>
        <li><code>start with</code> for partial leading match</li>
        <li><code>end with</code> for partial trailing match</li>
        <li><code>match regex</code> for regex match</li>
        <li><code>match xpath</code> for XML match</li>
        <li><code>match json path</code> for <Link to="https://github.com/json-path/JsonPath/blob/master/README.md">JSON match</Link>, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v1.4.0">v1.4.0</Link></li>
        <li><code>match template</code> for <Link to="/docs/template-matching">template</Link> match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
        <li><code>match template file</code> for <Link to="/docs/template-matching">template</Link> file match, since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v2.16.0">v2.16.0</Link></li>
      </ul>
    </li>
    <li><code>&lt;expression&gt;</code> is the expression to match (can be in <Link to="/docs/doc-strings">DocString</Link> position)</li>
    <li><code>&lt;timeoutSecs&gt;</code> is the timeout period in seconds</li>
    <li><code>&lt;wait|timeout&gt;</code> is either <code>wait</code> or <code>timeout</code></li>
  </ul>
  <p>Example</p>

  ```gherkin {4}
    Given the age dropdown can be located by id "challenger_age"
     When I navigate to "https://challengers.flood.io/step/2"
      And I select the 2nd option in the age dropdown
     Then the age dropdown value should be "18"
  ```

  Trimming and ignoring case (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.62.0">v3.62.0</Link>)
  <ul>
    <li>The <code>@Trim</code> annotation can be used on step to trim strings when comparing</li>
    <li>The <code>@IgnoreCase</code> annotation can be used on step to ignore case when comparing</li>
  </ul>

      <code>&lt;match&gt;</code> is the type of match to perform, one of:
      <ul>
        <li><code>be</code> for exact match</li>
        <li><code>contain</code> for partial match</li>
        <li><code>start with</code> for partial leading match</li>
