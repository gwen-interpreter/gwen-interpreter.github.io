<details id="i-format-number-from-pattern-to-pattern-as-name">
  <summary class="dsl">

  ```gherkin
  @Number I format <valueRef> from "<source-format>" to "<target-format>" as <name>
  ```
  <p>Converts the given numeric value to another format and stores the result in the given name in the global scope.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li>
      <code>&lt;valueRef&gt;</code> is the name of the binding containing the source numeric value
      <ul>
        <li>Can be the name of any binding that contains or resolves to a numeric value, including a web element in which case the text in the element will be dynamically retrieved and used.</li>
      </ul>
    </li>
    <li><code>&lt;source-format&gt;</code> is the source <Link to="https://docs.oracle.com/javase/8/docs/api/java/text/DecimalFormat.html#patterns">decimal number format pattern</Link> to convert from</li>
    <li><code>&lt;target-format&gt;</code> is the target <Link to="https://docs.oracle.com/javase/8/docs/api/java/text/DecimalFormat.html#patterns">decimal number format pattern</Link> to convert to</li>
    <li><code>&lt;name&gt;</code> is the name to bind the formatted numeric value to</li>
  </ul>
  <p>Example</p>

  ```gherkin {2}
    Given number is "1234"
     When @Number I format number from "#" to "$#,##0.00" as dollars
     Then dollars should be "$1,234.00"
  ```

  Since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v4.18.1">v4.18.1</Link>, Gwen also supports ordinal suffixes (1st, 2nd, 3rd, 4th, etc..)

  ```gherkin {2}
    Given number is "22"
     When @Number I format number from "#" to "#(st|nd|rd|th)" as ordingal number
     Then ordingal number should be "22nd"
  ```

  And (converting the other way)

  ```gherkin {2}
    Given ordingal number is "1st"
     When @Number I format ordingal number from "#(st|nd|rd|th)" to "00" as leading zero number
     Then leading zero number should be "01"
  ```
