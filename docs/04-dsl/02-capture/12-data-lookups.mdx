<details id="i-lookup-name-in-the-filepath-file-where-filter">
  <summary class="dsl">

  ```gherkin
  I lookup <name> in the "<filepath>" file where "<predicate>"
  ```
  <p>Looks up a named data value in a CSV or JSON file based on a predicate function and binds its value to the same name in the global scope.</p>
  </summary>
  <p>Where</p>
  <ul>
    <li><code>&lt;name&gt;</code>the name of the data to look up the value of and the name to bind it to</li>
    <li><code>&lt;filepath&gt;</code> the path to the CSV or JSON file to look up</li>
    <li>
      <code>&lt;predicate&gt;</code> is the javascript predicate expression that will return <code>true</code> when a match is found for a record or <code>false</code> otherwise (can be in <Link to="/docs/doc-strings">DocString</Link> position)
      Data fields can be referenced in the predicate using either:
      <ul>
        <li><code>$&#123;data.record.NAME&#125;</code> (since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.47.0">v3.47.0</Link>)</li>
        <li><code>$&#123;csv.record.NAME&#125;</code> for CSV data or <code>$&#123;json.record.NAME&#125;</code> for JSON data (for prior versions)</li>
      </ul>
    </li>
  </ul>

  The data value contained in the first record that satisifes the given predicate will be bound to the global scope. Otherwise if no records match, then no value will be bound.

  <p>Examples</p>

  ##### CSV Lookup

  Given the CSV file: lookup/StateCodes.csv

  ```csv {9}
  Code,Name
  ACT,Australian Capital Territory
  NSW,New South Wales
  NT,Northern Territory
  QLD,Queensland
  SA,South Australia
  TAS,Tasmania
  VIC,Victoria
  WA,Western Australia
  ```

  The following will bind "Western Australia" in the last record to <code>Name</code> in the global scope

  ```gherkin {1}
    When I lookup Name in the "lookup/StateCodes.csv" file where "'${data.record.Code}' == 'WA'"
    Then Name should be "Western Australia"
  ```

  ##### JSON Lookup
 
  > Since <Link to="https://github.com/gwen-interpreter/gwen-web/releases/tag/v3.47.0">v3.47.0</Link>

  Given the JSON file: lookup/StateCodes.json

  ```json {5}
  [
    { "Code": "ACT", "Name": "Australian Capital Territory" },
    { "Code": "NSW", "Name": "New South Wales" },
    { "Code": "NT",  "Name": "Northern Territory" },
    { "Code": "QLD", "Name": "Queensland" },
    { "Code": "SA",  "Name": "South Australia" },
    { "Code": "TAS", "Name": "Tasmania" },
    { "Code": "VIC", "Name": "Victoria" },
    { "Code": "WA",  "Name": "Western Australia" }
  ]
  ```

  The following will bind "Western Australia" in the last record to <code>Name</code> in the global scope

  ```gherkin {1}
    When I lookup Name in the "lookup/StateCodes.json" file where "'${data.record.Code}' == 'WA'"
    Then Name should be "Western Australia"
  ```
