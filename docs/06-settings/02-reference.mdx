# Settings Reference

## Core Settings

<details id="gwen-assertion-mode">
  <summary class="setting">

  ```properties
  gwen.assertion.mode
  ```
  <p>Controls whether or not hard, soft or sustained assertions are enabled.</p>
  </summary>
  <p>Supported values</p>
  <ul>
    <li><code>hard</code> to halt feature execution on the first assertion error and report failure</li>
    <li><code>soft</code> to collect all assertion errors and continue execution whilst reporting failures</li>
    <li><code>sustained</code> to collect all assertion errors and continue execution without reporting failures</li>
  </ul>
  Default value: <code>hard</code>
  <Tabs
  groupId="config"
  defaultValue="properties"
  values={[
    {label: 'gwen.properties', value: 'properties'}
  ]}>

  <TabItem value="properties">

  ```properties
    gwen.assertion.mode = hard
  ```

  </TabItem>

  </Tabs>

<details id="gwen-associative-meta">
  <summary class="setting">

  ```properties
  gwen.associative.meta
  ```
  <p>Controls whether or not same named feature and meta files in a directory are to be exclusively associated.</p>
  </summary>
  <p>Supported values</p>
  <ul>
    <li><code>true</code> to enable association
      <ul>
        <li>Gwen will load a discovered meta file only if it has the same name and is in the same directory as the current feature being executed.</li>
      </ul>
    </li>
    <li><code>false</code> to disable association
      <ul>
        <li>Gwen will unconditionally load every meta file discovered in the path to the current feature being executed.</li>
      </ul>
    </li>
  </ul>
  Default value: <code>false</code> (<code>true</code> will be the new default in the upcomming Gwen 3 release)
  <Tabs
  groupId="config"
  defaultValue="properties"
  values={[
    {label: 'gwen.properties', value: 'properties'}
  ]}>

  <TabItem value="properties">

  ```properties
    gwen.associative.meta = false
  ```

  </TabItem>

  </Tabs>

<details id="gwen-behavior-rules">
  <summary class="setting">

  ```properties
  gwen.behavior.rules
  ```
  ```properties
  gwen.behaviour.rules
  ```
  <p>Controls whether strict or lenient rules around Given-When-Then usage will be enforced for features.</p>
  </summary>
  <p>Supported values</p>
  <ul>
    <li><code>strict</code> to enforce the following strict rules:
      <ul>
        <li>Steps in scenarios and backgrounds must satisfy Given-When-Then order</li>
        <li>Givens must set context</li>
        <li>Whens must perform actions</li>
        <li>Thens must assert expectations</li>
        <li>Buts must perform assertions (and are permitted after Then steps)</li>
        <li>StepDefs tagged with
          <ul>
            <li><code>@Context</code> : can only be called by Givens</li>
            <li><code>@Action</code> : can only be called by Whens</li>
            <li><code>@Assertion</code> : can only be called by Thens or Buts</li>
          </ul>
        </li>
      </ul>
    </li>
    <li><code>lenient</code> to relax all strict rules</li>
  </ul>
  Default value: <code>lenient</code>
  <Tabs
  groupId="config"
  defaultValue="properties"
  values={[
    {label: 'gwen.properties', value: 'properties'}
  ]}>

  <TabItem value="properties">

  ```properties
    gwen.behavior.rules = lenient
  ```
  ```properties
    gwen.behaviour.rules = lenient
  ```

  </TabItem>

  </Tabs>

  <div class="grid-3">

<details id="gwen-rampup-interval-seconds">
  <summary class="setting">

  ```properties
  gwen.rampup.interval.seconds
  ```
  <p>Sets the ramp up interval for staggered parallel execution.</p>
  </summary>
  <p>Supported values</p>
  <ul>
    <li>A positive integer denoting the number of seconds to wait before executing successive features in parallel</li>
    <li>Zero for no inverval or delay between successive feature executions</li>
  </ul>
  Default value: <code>0</code>
  <p>
    <br/>Example: A 5 second interval would stagger executions like this over 4 cores:<br/><br/>

  ```
    time  00   05   10   15   seconds
          │    │    │    │
  core 1  │<── feature 1a ──>|<── feature 1b ──> ..
  core 2  │    │<── feature 2a ──>|<── feature 2b ──> ..
  core 3  │    │    │<── feature 3a ──>|<── feature 3b ──> ..
  core 4  │    │    │    │<── feature 4a ──>|<── feature 4b ──> ..
  ```

  </p>
  <Tabs
  groupId="config"
  defaultValue="properties"
  values={[
    {label: 'gwen.properties', value: 'properties'}
  ]}>

  <TabItem value="properties">

  ```properties
    gwen.rampup.interval.seconds = 0
  ```
